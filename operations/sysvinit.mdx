Manage sysvinit services (`/etc/init.d`).

## Functions

### `sysvinit.service`

Manage the state of SysV Init (/etc/init.d) services.

```python
sysvinit.service(
    service,
    running=True,
    restarted=False,
    reloaded=False,
    enabled=None,
    command=None,
)
```

<ParamField name="service" type="str" required>
  Name of the service to manage
</ParamField>

<ParamField name="running" type="bool" default="True">
  Whether the service should be running
</ParamField>

<ParamField name="restarted" type="bool" default="False">
  Whether the service should be restarted
</ParamField>

<ParamField name="reloaded" type="bool" default="False">
  Whether the service should be reloaded
</ParamField>

<ParamField name="enabled" type="bool" default="None">
  Whether this service should be enabled/disabled
</ParamField>

<ParamField name="command" type="str" default="None">
  Command (eg. reload) to run like: `/etc/init.d/<service> <command>`
</ParamField>

<Note>
**Enabled:** Because managing /etc/rc.d/X files is a mess, only certain Linux distributions support enabling/disabling services:

- Ubuntu/Debian (`update-rc.d`)
- Fedora/RHEL (`chkconfig`)
- Gentoo (`rc-update`)

For other distributions and more granular service control, see the `sysvinit.enable` operation.
</Note>

### `sysvinit.enable`

Manually enable /etc/init.d scripts by creating /etc/rcX.d/Y links.

```python
sysvinit.enable(
    service,
    start_priority=20,
    stop_priority=80,
    start_levels=(2, 3, 4, 5),
    stop_levels=(0, 1, 6),
)
```

<ParamField name="service" type="str" required>
  Name of the service to enable
</ParamField>

<ParamField name="start_priority" type="int" default="20">
  Priority to start the service
</ParamField>

<ParamField name="stop_priority" type="int" default="80">
  Priority to stop the service
</ParamField>

<ParamField name="start_levels" type="tuple" default="(2, 3, 4, 5)">
  Which runlevels should the service run when enabled
</ParamField>

<ParamField name="stop_levels" type="tuple" default="(0, 1, 6)">
  Which runlevels should the service stop when enabled
</ParamField>

## Examples

```python
from pyinfra.operations import sysvinit

# Restart and enable rsyslog
sysvinit.service(
    name="Restart and enable rsyslog",
    service="rsyslog",
    restarted=True,
    enabled=True,
)

# Finer control on which runlevels rsyslog should run
sysvinit.enable(
    name="Finer control on which runlevels rsyslog should run",
    service="rsyslog",
    start_levels=(3, 4, 5),
    stop_levels=(0, 1, 2, 6),
)
```
