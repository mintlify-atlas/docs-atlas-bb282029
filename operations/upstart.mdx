Manage upstart services.

## Functions

### `upstart.service`

Manage the state of upstart managed services.

```python
upstart.service(
    service,
    running=True,
    restarted=False,
    reloaded=False,
    command=None,
    enabled=None,
)
```

<ParamField name="service" type="str" required>
  Name of the service to manage
</ParamField>

<ParamField name="running" type="bool" default="True">
  Whether the service should be running
</ParamField>

<ParamField name="restarted" type="bool" default="False">
  Whether the service should be restarted
</ParamField>

<ParamField name="reloaded" type="bool" default="False">
  Whether the service should be reloaded
</ParamField>

<ParamField name="command" type="str" default="None">
  Custom command to pass like: `/etc/rc.d/<service> <command>`
</ParamField>

<ParamField name="enabled" type="bool" default="None">
  Whether this service should be enabled/disabled on boot
</ParamField>

<Note>
**Enabling/disabling services:** Upstart jobs define runlevels in their config files - as such there is no way to edit/list these without fiddling with the config. So pyinfra simply manages the existence of a `/etc/init/<service>.override` file, and sets its content to "manual" to disable automatic start of services.
</Note>

## Examples

```python
from pyinfra.operations import upstart

# Manage an Upstart service
upstart.service(
    name="Ensure nginx is running",
    service="nginx",
    running=True,
    enabled=True,
)
```
